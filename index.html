This is my crack at a JavaScript-backed implementation of John Conway's ageless Game of Life.

    The rules---as much for my benefit as yours---are as follows: Every generation
  • Any live cell with fewer than two live neighbors dies
  • Any live cell with more than three live neighbors dies
  • Any live cell with two or three live neighbors survives
  • Any dead cell with exactly three live neighbors becomes alive
 
    The modes of interaction with this board are as follows:
  • Left-click any empty cell to make it alive
  • Left-click any live cell to kill it off
  • Left-click and drag anywhere on the board to move the view frustum
  • TODO: Right-click movement. Mouse-wheel zooming.

I've put this whole thing for my own edification. It was an experiment designed solely to glean a familiarity with CoffeeScript (and, as a result, JavaScript), HTML5 and CSS3. I wanted to do as much of the formatting---structure, typography, visual flare---in HTML and CSS less so that I can learn the old way of putting web pages together, but so that I know what to look for in the "right" way.

Ahh fuck it. I'll add this content in when I have the structure fleshed out.

It should be noted that the grid I've implemented is (theoretically) infinite, so any gliders you loose will just keep on going. This shouldn't be a problematic memory leak (assuming you don't stock up on Glider Guns) as the underlying model only tracks live cells. This does, however, suggest a rather inefficient implementation---it is a web-app, after all---so don't expect anything extraordinary.
